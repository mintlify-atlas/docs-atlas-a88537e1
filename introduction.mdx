# Meet Claudio

Claudio is an AI-powered productivity assistant built specifically for PropHero's Product & Technology team. It connects your development tools—ClickUp, GitHub, Slack, Google Docs, and more—to automate repetitive tasks and let you focus on what matters: building great products.

<Info>
  **Built for async-first teams**: Interact with Claudio via Terminal, Telegram, Slack, or Web Dashboard—wherever you work.
</Info>

## What can Claudio do?

Claudio connects to your tools through MCPs (Model Context Protocol) servers, giving it hands-on access to:

### Product management
- **Create Initiatives, Epics, and User Stories** in ClickUp with proper templates
- **Query tasks and track progress** across sprints
- **Generate sprint reports** automatically from ClickUp data

### Development workflows
- **Check GitHub PRs and issues** without leaving your chat
- **Create and update GitHub issues** with context from ClickUp
- **Execute terminal commands** in your workspace

### Team communication
- **Post updates to Slack channels** for daily standups and reports
- **Create and share Google Docs** with team meeting notes
- **Send automated reports** via email from scheduled scripts

### Data and insights
- **Query BigQuery** for funnel analysis and metrics
- **Access Google Sheets** for performance dashboards
- **Retrieve meeting notes** from Granola transcriptions

<Note>
  Claudio follows the "Execution > Explanation" principle: it prioritizes doing over talking, always confirms before creating, and provides concise summaries.
</Note>

## How it works

Claudio follows a simple architecture:

```
Channels (Mouths) → Brain (Docs/Workflows) → Hands (MCPs)
     ↓                      ↓                      ↓
  Terminal            CLAUDE.md              ClickUp MCP
  Telegram           Workflows               GitHub MCP
  Slack              Templates               Slack MCP
  Web                                         Google Docs MCP
```

<Steps>
  <Step title="You send a request">
    From any channel: "Create a user story for login feature"
  </Step>
  <Step title="Claudio reads context">
    Checks CLAUDE.md for identity and rules, looks up relevant templates and workflows
  </Step>
  <Step title="Claudio executes with MCPs">
    Uses ClickUp MCP to query Initiative, creates User Story with proper template
  </Step>
  <Step title="You get results">
    Receives link to created User Story with full context and proper formatting
  </Step>
</Steps>
