Get Claudio running and execute your first productivity workflow in minutes. This guide covers installation, MCP configuration, and usage across all channels.

## Prerequisites

Before you begin, ensure you have:

- **Python 3.10+** and **Node.js 18+** installed
- **[Claude Code CLI](https://docs.anthropic.com/en/docs/claude-code)** (for bot channels and automation scripts)
- **Cursor IDE** (optional, for direct terminal usage)

<Note>
  This guide assumes you have a PropHero team account with access to ClickUp, Slack, and Google Workspace.
</Note>

## Installation

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/cuba1511/claudio.git
    cd claudio
    ```
  </Step>

  <Step title="Set up Python environment">
    ```bash
    python3 -m venv venv
    source venv/bin/activate  # On Windows: venv\Scripts\activate
    pip install -r requirements.txt
    ```
  </Step>

  <Step title="Configure environment variables">
    Copy the example environment file and fill in your values:

    ```bash
    cp .env.example .env
    ```

    Edit `.env` and set:

    ```bash
    WORKSPACE_PATH=/path/to/claudio
    CLAUDE_CLI_PATH=claude
    ALLOWED_TOOLS=*
    SKIP_PERMISSIONS=true
    COMMAND_TIMEOUT=1800
    ```

    <Tip>
      See the [Environment Configuration](/configuration/environment) guide for detailed explanations of all variables.
    </Tip>
  </Step>

  <Step title="Configure MCPs">
    Set up your MCP integrations in `~/.cursor/mcp.json` (for Cursor) or `~/.claude.json` (for CLI):

    ```json
    {
      "mcpServers": {
        "clickup": {
          "command": "npx",
          "args": ["-y", "@taazkareem/clickup-mcp-server@latest"],
          "env": {
            "CLICKUP_MCP_LICENSE_KEY": "your-license-key",
            "CLICKUP_API_KEY": "your-api-key",
            "CLICKUP_TEAM_ID": "your-team-id"
          }
        },
        "slack": {
          "command": "npx",
          "args": ["-y", "slack-mcp-server@latest", "--transport", "stdio"],
          "env": {
            "SLACK_MCP_XOXC_TOKEN": "xoxc-your-token",
            "SLACK_MCP_XOXD_TOKEN": "xoxd-your-token"
          }
        }
      }
    }
    ```

    <Note>
      For complete MCP configuration instructions, see the [MCP Setup](/configuration/mcp-setup) guide.
    </Note>
  </Step>
</Steps>

## Choose your channel

Claudio supports multiple interaction channels. Pick the one that fits your workflow:

<Tabs>
  <Tab title="Terminal (Cursor)">
    **Best for**: Direct usage during development

    <Steps>
      <Step title="Open Cursor IDE">
        Open the Claudio project directory in Cursor:
        ```bash
        cursor .
        ```
      </Step>

      <Step title="Verify MCP configuration">
        Ensure your `~/.cursor/mcp.json` is configured with the MCPs from the previous step.
      </Step>

      <Step title="Restart Cursor">
        Restart Cursor to load the MCP configuration.
      </Step>

      <Step title="Start chatting">
        Open the Claude chat panel in Cursor and start asking Claudio to help you:
        
        ```
        Create a user story in ClickUp for adding dark mode to the settings page
        ```
      </Step>
    </Steps>
  </Tab>

  <Tab title="Telegram">
    **Best for**: Mobile access and quick commands on the go

    <Steps>
      <Step title="Create a Telegram bot">
        Message [@BotFather](https://t.me/BotFather) on Telegram and create a new bot with `/newbot`. Save your bot token.
      </Step>

      <Step title="Get your user ID">
        Message [@userinfobot](https://t.me/userinfobot) to get your Telegram user ID.
      </Step>

      <Step title="Configure the bot">
        Create `channels/telegram/.env`:
        
        ```bash
        TELEGRAM_BOT_TOKEN=your-bot-token
        ALLOWED_USER_IDS=your-user-id
        WORKSPACE_PATH=/path/to/claudio
        OPENAI_API_KEY=sk-your-openai-key  # For voice transcription
        ```
      </Step>

      <Step title="Install dependencies and start">
        ```bash
        cd channels/telegram
        pip install -r requirements.txt
        ./start.sh
        ```
      </Step>

      <Step title="Message your bot">
        Find your bot on Telegram and send `/start` to begin chatting.
      </Step>
    </Steps>

    <Tip>
      See the [Telegram Channel](/channels/telegram) guide for advanced configuration and commands.
    </Tip>
  </Tab>

  <Tab title="Slack">
    **Best for**: Team collaboration and shared workflows

    <Steps>
      <Step title="Create a Slack app">
        1. Go to [api.slack.com/apps](https://api.slack.com/apps)
        2. Click **Create New App** → **From scratch**
        3. Name it "Claudio" and select your workspace
      </Step>

      <Step title="Configure OAuth scopes">
        Under **OAuth & Permissions**, add these Bot Token Scopes:
        - `chat:write`
        - `app_mentions:read`
        - `im:history`
        - `im:read`
        - `im:write`
      </Step>

      <Step title="Enable Socket Mode">
        1. Under **Socket Mode**, enable it and generate an app-level token
        2. Under **Event Subscriptions**, enable events and subscribe to:
           - `app_mention`
           - `message.im`
      </Step>

      <Step title="Install to workspace">
        Click **Install to Workspace** and authorize the app.
      </Step>

      <Step title="Configure the bot">
        Create `channels/slack/.env`:
        
        ```bash
        SLACK_BOT_TOKEN=xoxb-your-bot-token
        SLACK_APP_TOKEN=xapp-your-app-token
        SLACK_ALLOWED_USER_IDS=U1234567890
        WORKSPACE_PATH=/path/to/claudio
        ```
      </Step>

      <Step title="Install dependencies and start">
        ```bash
        cd channels/slack
        pip install -r requirements.txt
        ./start.sh
        ```
      </Step>

      <Step title="Message the bot">
        Send a DM to Claudio in Slack or mention `@Claudio` in any channel.
      </Step>
    </Steps>

    <Tip>
      See the [Slack Channel](/channels/slack) guide for slash commands and advanced usage.
    </Tip>
  </Tab>

  <Tab title="Web Dashboard">
    **Best for**: Monitoring MCPs and interactive chat with streaming responses

    <Steps>
      <Step title="Install dependencies">
        ```bash
        cd channels/web
        pip install -r requirements.txt
        ```
      </Step>

      <Step title="Start the server">
        ```bash
        ./start.sh
        # Or for development with hot reload:
        ./start.sh --dev
        ```
      </Step>

      <Step title="Open the dashboard">
        Navigate to [http://localhost:8000](http://localhost:8000) in your browser.
      </Step>

      <Step title="Chat with Claudio">
        Click **Chat** in the navigation and start typing your requests.
      </Step>
    </Steps>

    <Tip>
      The web dashboard provides real-time MCP health monitoring at [http://localhost:8000/api/mcps/health](http://localhost:8000/api/mcps/health).
    </Tip>
  </Tab>
</Tabs>

## Your first workflow

Let's create a User Story in ClickUp using the ClickUp MCP integration:

<Steps>
  <Step title="Start a conversation">
    Using your preferred channel (Terminal, Telegram, Slack, or Web), ask Claudio:

    ```
    Create a user story for adding a password reset feature to the login page
    ```
  </Step>

  <Step title="Claudio gathers context">
    Claudio will:
    1. Read the parent Initiative or Epic for context
    2. Use the User Story template from `docs/integrations/clickup/templates/`
    3. Extract relevant information about the project
  </Step>

  <Step title="Task is created">
    Claudio will use the ClickUp MCP to create a task with:
    - Title and description
    - Acceptance criteria
    - Technical notes
    - Business value
    - Assignment to the Sprint Backlog list
  </Step>

  <Step title="Get the link">
    Claudio responds with a link to the newly created ClickUp task:
    
    ```
    ✅ Created User Story: "Add password reset feature"
    📎 https://app.clickup.com/t/901213056238/TASK-123
    
    💡 Next: Review the task and assign it to a sprint
    ```
  </Step>
</Steps>

<Info>
  This workflow uses multiple tools: `get_workspace_hierarchy` (context), `get_task` (parent info), and `create_task` (creation).
</Info>

## Troubleshooting

<AccordionGroup>
  <Accordion title="MCP not found or connection error">
    **Cause**: The MCP server isn't configured or credentials are invalid.

    **Solution**:
    1. Verify your MCP configuration in `~/.cursor/mcp.json` or `~/.claude.json`
    2. Check that all environment variables (API keys, tokens) are set correctly
    3. Restart Cursor or the bot to reload the configuration
    4. Test MCP health at `http://localhost:8000/api/mcps/health` (web dashboard)
  </Accordion>

  <Accordion title="Permission denied errors">
    **Cause**: `SKIP_PERMISSIONS` is not enabled or tool permissions are restricted.

    **Solution**:
    Set in your `.env`:
    ```bash
    SKIP_PERMISSIONS=true
    ALLOWED_TOOLS=*
    ```
  </Accordion>

  <Accordion title="Bot not responding">
    **Cause**: User not authorized, bot process crashed, or rate limit exceeded.

    **Solution**:
    1. Check that your user ID is in `ALLOWED_USER_IDS` (Telegram) or `SLACK_ALLOWED_USER_IDS` (Slack)
    2. Check bot logs in `channels/{telegram|slack}/` or `scripts/logs/`
    3. Verify the bot process is running: `ps aux | grep bot.py`
    4. Restart the bot: `./channels/{channel}/start.sh`
  </Accordion>

  <Accordion title="Command timeout">
    **Cause**: Task took longer than `COMMAND_TIMEOUT` (default 30 minutes).

    **Solution**:
    Increase the timeout in `.env`:
    ```bash
    COMMAND_TIMEOUT=3600  # 1 hour
    ```
  </Accordion>
</AccordionGroup>

<Warning>
  If you encounter authentication errors with MCPs (ClickUp, Slack, Google), double-check that your API keys and tokens are valid and have the required permissions.
</Warning>
